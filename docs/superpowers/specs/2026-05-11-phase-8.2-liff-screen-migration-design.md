## Goals

Port the remaining employee screens (Request, History, Profile) into the root LIFF app, wire up a shared LIFF shell (auth gate + offline banner + tab bar), and replace employee-specific patterns (PIN, Web Share, logout) with LIFF-native equivalents.

---

## Route Structure

Create a `(liff)` route group. Group parentheses are transparent to URLs.

```
app/
  (liff)/
    layout.tsx          ← LIFFAuthGate + LiffShell (mounts once)
    page.tsx            ← moved from app/page.tsx; LiffHomePage with its own gate removed
    request/
      page.tsx          ← new; renders LiffRequestPage
    history/
      page.tsx          ← new; renders LiffHistoryPage
    profile/
      page.tsx          ← new; renders LiffProfilePage
  [locale]/             ← unchanged (HR + legacy employee routes)
  layout.tsx            ← unchanged (HTML/body root)
```

**URLs:** `/`, `/request`, `/history`, `/profile` — unchanged from what `LiffHomePage` already links to.

## LiffShell

**File:** `components/liff-shell.tsx`

Thin layout component used only by `(liff)/layout.tsx`.

```
LiffShell
  ├── LiffOfflineBanner
  ├── <main className="flex-1 overflow-auto pb-16"> children </main>
  └── LiffBottomTabBar
```

The `(liff)/layout.tsx` is an async server component:

```tsx
export default async function LiffLayout({ children }) {
  const messages = await getMessages()  // falls back to 'th' via i18n/request.ts
  return (
    <NextIntlClientProvider messages={messages}>
      <ToastProvider>
        <LIFFAuthGate>
          <LiffShell>{children}</LiffShell>
        </LIFFAuthGate>
      </ToastProvider>
    </NextIntlClientProvider>
  )
}
```

`NextIntlClientProvider` is required here because `LiffRequestPage`, `LiffHistoryPage`, and `LiffProfilePage` use `useTranslations()`. `getMessages()` falls back to `defaultLocale: 'th'` for routes without a locale segment (handled by `i18n/request.ts`).

`LiffHomePage` loses its own `<LIFFAuthGate>` wrapper after the move — the layout provides it. `useLiffProfile()` works anywhere inside the group via the existing context.

## LiffOfflineBanner

**File:** `components/liff-offline-banner.tsx`

Two distinct states, checked in order:

| Condition | Banner content |
|-----------|---------------|
| `!navigator.onLine` | WifiOff icon + "ไม่มีอินเทอร์เน็ต" |
| `navigator.onLine && !liff.isInClient() && !LIFF_MOCK` | ExternalLink icon + "เปิดใน LINE เพื่อใช้งานได้ครบถ้วน" |
| Both online + in client | `null` |

`liff.isInClient()` is synchronous after `liff.init()` has resolved. The gate has already completed by the time `LiffShell` renders, so `loadLiffClient()` can be called directly (same pattern as `liff-auth-gate.tsx`).

The original `components/employee/offline-banner.tsx` is left unchanged.

## LiffRequestPage

**File:** `components/liff-request-page.tsx`

Adapted from `EmployeeRequestPage`. Three steps, same `StepIndicator`.

### Step 1 — Amount + Reason
Unchanged from `EmployeeRequestPage`: available balance display, quick-amount buttons, custom amount input, reason chips, "Next →" button.

### Step 2 — Confirmation
Summary card is unchanged. The PIN pad is replaced with a mock OTP field:

- Label: "รหัสยืนยัน (OTP)"
- `<input type="text" inputMode="numeric" maxLength={6} />`
- Helper text below input: "กรอก 000000 เพื่อยืนยัน" (removed when backend is wired)
- Confirm button enables when input length === 6
- On submit: accept `"000000"`, show inline error "รหัสไม่ถูกต้อง" on any other value, advance to step 3 on success

### Step 3 — Success
Confetti animation unchanged. Share button:

- Calls `liff.shareTargetPicker()` when `liff.isInClient()` is true
- Button hidden when `!liff.isInClient()`
- Uses `loadLiffClient()` to get the liff instance

## LiffHistoryPage

**File:** `components/liff-history-page.tsx`

Structurally identical to `EmployeeHistoryPage` (tabs, accordion cards, status badges).

### LIFF-specific changes

**Deep-link support:** On mount, read `?id=<requestId>` via `useSearchParams()`. If present and a matching request exists, set that request as the initially expanded accordion card.

**Removed:** "Export pay slip" PDF button inside the expanded accordion detail — not relevant in LIFF context.

---

## LiffProfilePage

**File:** `components/liff-profile-page.tsx`

Adapted from `EmployeeProfilePage`.

### Changes from employee version

**Header avatar:** Use `useLiffProfile()?.pictureUrl` via `<Image>` (same pattern as `LiffHomePage`). Fall back to initials avatar when `pictureUrl` is absent.

**LINE display name:** Show `useLiffProfile()?.displayName` as a secondary line beneath the employee name.

**Removed:** `<LogOut>` button at the bottom.

**Added:** "ยกเลิกการเชื่อมต่อ LINE" (Unlink LINE account) button at the bottom — destructive style (red text, border). On tap:
1. Read current `userId` from `useLiffProfile()`
2. Remove that key from `payday-liff-employee-links` in `localStorage`
3. Call `window.location.reload()` → user is returned to the linking flow in `LIFFAuthGate`

Notification toggles, bank account sheet, language switcher, and EWA limit section are unchanged.
